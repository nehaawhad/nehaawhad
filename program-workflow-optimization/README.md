# Program Workflow Optimization

This case study is based on work I did during my Program Management internship at CompuCom.

The goal was to improve how program tasks were tracked and executed across multiple teams.

---

## The Problem

Program execution involved multiple teams such as procurement, engineering, and operations.

Because of several handoffs between teams, tickets sometimes faced delays in verification and execution.

---

## What I Worked On

My work focused on understanding the workflow and making it more structured.

• Documented program workflows

• Created process documentation and SOPs

• Built swimlane diagrams to clarify responsibilities

• Developed templates to standardize ticket verification

---

## Tracking Performance

To better understand performance, dashboards were created using SQL, Excel, and Power BI.

These dashboards tracked:

• ticket volumes
• SLA performance
• aging tickets

This helped teams identify delays and prioritize tasks more effectively.

---

## Outcome

Introducing structured workflows and clearer documentation helped improve coordination between teams and reduced verification delays.

---

## Key Takeaway

Small improvements in process documentation and visibility can significantly improve program execution across teams.
